# Feature Writing Rules

- Every PRD must reference at least one persona from `05-user-personas/`.
- Success metrics must use real baselines from existing files, not invented targets.
- Include a "What we're NOT building" section to define scope boundaries.
- Reference `04-strategy/okrs-q2-2026.md` to show how the feature connects to company goals.
- User stories must follow: "As a [persona name], I want [action], so that [benefit]."
