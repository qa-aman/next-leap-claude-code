# Layer 3 Reference - SUCCESs Quality Checklist

From "Made to Stick" (Chip Heath and Dan Heath). This is not a writing structure, it is a quality gate. Run it on each finished output before delivering.

## The Six Checks

Score each output pass/fail on all six. One line of evidence per check.

### 1. Simple
One core idea per output. If the blurb tries to sell two benefits, it fails. Test: can the reader repeat the point in one sentence after one read?

### 2. Unexpected
Does the opener break the pattern the reader expects? "We're excited to announce" is the expected pattern, so it fails. A surprising number or an uncomfortable truth about their current workflow passes.

### 3. Concrete
Specific nouns and numbers, not abstractions. "Faster follow-ups" fails. "A drafted follow-up email before you're back at your desk" passes. Every number must come from a repo file.

### 4. Credible
Is there proof a skeptic would accept? Real metrics with sources, named plan tiers, concrete launch dates. Vague superlatives fail this check. If no honest proof exists for a claim, cut the claim rather than inflate it.

### 5. Emotional
Does it touch what the reader feels about the problem - the embarrassment of a dropped action item, the relief of trusting the summary? Pure feature listing fails.

### 6. Stories
Is there a micro-narrative arc (before-state, turn, after-state)? Even the 90-character banner can imply one. A static description fails.

## Scoring Honestly

The point of this layer is to catch your own weak copy, not to rubber-stamp it. If you are unsure whether a check passes, it fails. Short formats (banner, social) get judged on compressed versions of the checks - a banner is Simple by necessity, so judge it harder on Concrete and Unexpected.

Rewrite rule: 4/6 or below means one targeted rewrite of the failing checks, then rescore. Never more than one rewrite pass - if it still fails, flag the weak output to the user with the failing checks named.
