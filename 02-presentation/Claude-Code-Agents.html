PART 1 · WHY IT MATTERS

A Chatbot Answers. An Agent Gets It Done.

  • ChatGPT in 2023: you ask, it replies, you copy-paste the answer.
  • An agent: you give it a goal, it plans, it acts, it iterates.
  • The shift is from AI that talks to AI that takes actions in the world.
  • Booking flights, writing code, running research - without you in every step.
  • This is the difference between a tool and a teammate.
The gap between people who get this and people who don't is about to get expensive.

Prompt vs Agent - The Driver Analogy

PROMPT = STUDENT DRIVER
You sit next to them.
  • You guide every turn.
  • You correct mistakes in real time.
  • You stay alert the whole drive.

AGENT = HIRED DRIVER

  • It picks the route.
  • It handles traffic and detours.

PART 2 · HOW THEY WORK

An AI Agent: The Simple Definition

A system that uses an LLM to decide what to do next, takes that action, and uses the result to decide what to do after that.
  • The LLM is the brain. It reasons, it picks the next step.
  • Tools are the hands. The agent calls them to act on the world.
  • The loop is what matters. Decide, do, observe, decide again.

Workflow vs Agent - Grocery on Friday

Same job: every Friday, check prices, build my list, place the order.
WORKFLOW
Designed to be obedient.

AGENT

  • Adjusts quantities for six.
  • Rebuilds the order around context.

PART 3 · WHEN TO USE

The ARR Test - Is This an Agent or a Prompt?

If a task is Autonomous, Recurring, and Reviewable - it's an agent. If it needs live judgment, happens once, or can't be reviewed - keep it a prompt.
A
Autonomous
Can run without you stepping in at every decision.
R
Recurring
Happens again - daily, weekly, on a trigger.
R
Reviewable
You can tell if the output is good or bad.

PART 4 · WHERE THEY BREAK

An Agent Is a Multiplier, Not Magic

An agent will do the wrong thing faster and with more confidence than you ever could. It doesn't fix bad thinking - it formalizes it.
  • Most AI problems are human problems in disguise.
  • An agent is a mirror. It reflects the quality of your thinking back.
  • Vague goals + sloppy directions = the car hits the tree, faster.
  • Usually the agent didn't fail. The human was vague.

PART 5 · HANDS ON WITH CLAUDE CODE

Skill vs Subagent - What's the Difference?

Think of it like a kitchen. A Skill is the recipe. A Subagent is the cook.
SKILL
The recipe.
  • Instructions Claude follows when the moment fits.
  • You write it once. Reuse it every time.
  • Lives in your project - the whole team gets it.
  • Best for: playbooks, frameworks, checklists.
  • Example: 'Write a PRD the way our team does.'
SUBAGENT
The cook.
  • A worker Claude sends off to do a job.
  • Works in its own room - no mess in your chat.
  • Comes back with just the answer.
  • Best for: research, audits, deep file reading.
  • Example: 'Read all 12 interviews. Tell me the top 3.'
You give Claude the recipe. Claude can either follow it itself, or hand it to a cook.

PART 5 · HANDS ON WITH CLAUDE CODE

A Subagent Is Just a Markdown File

  • YAML frontmatter at the top - name, description, tools, model.
  • Markdown body becomes the system prompt.
  • Only name and description are required.
  • Store in .claude/agents/ (project) or ~/.claude/agents/ (you).
  • Use /agents in Claude Code to create one interactively.

REFERENCE · GLOSSARY

Glossary - Terms You'll Hear All Workshop

Agent
A Claude instance that decides its own next step and uses tools to act.
Subagent
A separate Claude instance dispatched by a main agent, with its own context.
LLM
The language model underneath (Sonnet, Opus, Haiku) that does the reasoning.
Tool use
An agent calling external functions (web search, file read, an API).
Orchestrator
An agent whose job is to delegate to other agents and combine results.
Fan-out / fan-in
Dispatch many subagents in parallel, then merge what they return.
Context window
The limited memory budget each agent works with per turn.
Hallucination
When an agent confidently outputs something that is not true.
Knowing the names makes the agent traces readable.

REFERENCE · FAQ

Five Questions You'll Be Asked Back at Work

1
Skill vs Subagent - what is the practical difference?
Per docs: a Skill is an instruction file. A Subagent is a specialized assistant that runs in its own context window with its own system prompt, tools, and permissions. Different mechanics.
2
Do agents still hallucinate?
Yes - any LLM-based agent can. Docs recommend isolating side work in Subagents (separate context) and reviewing diffs before accepting. Plan mode inspects the plan before any edit lands.
3
How do I keep agent costs from blowing up?
Docs: route routine tasks to faster, cheaper models like Haiku via Subagents. Reserve Opus for hard reasoning. Subagents preserve main context, avoiding repeat setup. Watch the token counter.
4
One big agent or many small subagents in parallel?
One main agent for sequential steps that depend on each other. Delegate side work to Subagents when it would otherwise flood the main context with file reads, logs, or search output.
5
Can a Claude-built agent run in production?
Production use needs monitoring, retries, rate-limit handling, and a budget cap. For internal tools and overnight batch jobs, agents are stable today. Plan and review diffs first.
