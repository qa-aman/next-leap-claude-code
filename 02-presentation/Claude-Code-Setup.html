PART 1 · INSTALL

Mac: One Command in Terminal

  • Open Terminal: Cmd+Space, type Terminal, press Enter
  • Paste and run the install command (shown right)
  • Wait for the success message — takes about 30 seconds
  • The native installer needs no Node.js or npm
  • Updates happen automatically in the background
  • Homebrew alternative: brew install --cask claude-code
# Mac / Linux / WSL
curl -fsSL \
  https://claude.ai/install.sh \
  | bash

# Homebrew (manual updates)
brew install --cask claude-code

# Verify
claude --version
This single command replaces a half-hour setup wizard — one line, done.

PART 1 · INSTALL

Windows: Three Steps in PowerShell

  • Open PowerShell (Win+S, type PowerShell). Pick the entry without (x86).
  • No need to run as Administrator.
  • Step 1: Paste the irm command shown right. Wait for success.
  • Step 2: In the same window, paste the SetEnvironmentVariable line to add Claude to your User PATH.
  • Step 3: Close PowerShell. Open a fresh one. Run claude --help.
# Step 1: install
irm https://claude.ai/install.ps1 `
  | iex

# Step 2: add to PATH
[Environment]::
SetEnvironmentVariable("PATH",
  [Environment]::
  GetEnvironmentVariable(
    "PATH", "User")
  + ";$HOME\.local\bin",
  "User")

# Step 3: new PowerShell
claude --help
Git for Windows is optional — PowerShell-only is enough for the workshop.

PART 1 · INSTALL

Five Errors You Will See — and Fixes

1
claude: command not found
Close and reopen terminal. Or add export PATH=$HOME/.local/bin:$PATH to ~/.zshrc
2
Browser does not open on login
Press the letter c in the terminal to copy the login URL, then paste it manually
3
Subscription required / auth fails
You need Claude Pro, Max, Teams, or Enterprise. The free Claude.ai plan does not include Claude Code
4
Git Bash not found (Windows only)
Install Git for Windows from git-scm.com/downloads/win — Claude Code needs it on native Windows
5
PowerShell execution policy error
Run: Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned, then retry

PART 2 · FIRST LAUNCH

Login Flow: Five Steps in 60 Seconds

1
Run claude in your terminal
You will see a welcome screen. It prompts you to log in.
2
Select 'Claude account with subscription'
Not the API key option. OAuth — a browser-based login — is simpler and what most people want.
3
Browser opens. Log in with your Claude.ai account.
If the browser does not open automatically, press the letter c to copy the URL and paste it yourself.
4
Return to terminal. Pick a text style.
On first launch, Claude asks: light or dark terminal theme. Pick what matches your background.
5
You are in. Credentials are saved.
You will not log in again unless you run /logout. On Mac, credentials live in the macOS Keychain.
OAuth means you never paste an API key — it is the same login flow as signing into Google.

PART 2 · FIRST LAUNCH

Three Ways to Check Your Install Is Healthy

claude --version
Run in terminal before starting a session
Prints the installed version number. If you see 'command not found', your PATH needs updating.
claude doctor
Run in terminal before starting a session
Runs full diagnostics. Checks install, subscription status, network connectivity, and system config. If something is wrong, it tells you what and why.
/doctor
Run inside an active Claude session
Same diagnostics as claude doctor but runs from within the interactive session (REPL). Use it to re-check status after logging in.
Run claude doctor whenever something feels off — it is the health check that tells you exactly what to fix.

PART 3 · CORE COMMANDS

Four Commands Every Session Uses

/help
Show all available commands and slash commands
Type this any time you are inside a Claude session. Lists every command, including skills you have installed.
/clear
Clear the conversation history
Wipes the current context window (the conversation Claude is holding in memory). Does not delete files. Use when Claude is confused or when you want a fresh start on a new topic.
/exit
Exit Claude Code (or use Ctrl+D)
Closes the interactive session cleanly. Your files and settings are untouched.
ESC
Interrupt — stop the current response immediately
Press Escape while Claude is generating to stop it mid-sentence. Useful when Claude goes in the wrong direction or when a command is taking too long.
ESC is your undo button — press it the moment Claude starts going somewhere wrong, before it writes a file you did not want.

PART 3 · CORE COMMANDS

Plan Mode: Claude Shows Its Work Before Acting

DEFAULT MODE
Claude acts immediately
  • You say 'update this doc'
  • Claude edits the file right away
  • You see changes after the fact
  • Good for quick, trusted tasks
PLAN MODE (Shift+Tab or /plan)
Claude proposes first, then waits
  • You say 'update this doc'
  • Claude describes what it will do
  • You review and approve
  • Then Claude acts — nothing before that
Use plan mode when stakes are high — before Claude touches a live document or an important file.

CLAUDE.md: The File Claude Reads Before Every Session

  • A plain text file you write once, Claude reads every time
  • Lives at the root of your project folder
  • Loaded automatically — you never have to re-explain context
  • Think of it as a briefing doc that survives every session
  • Without it: Claude has no memory of your project
  • With it: Claude knows your stack, rules, and role from word one
project-root/
  CLAUDE.md   <- reads first
  src/
  outputs/
  .claude/
    settings.json
    rules/

~/.claude/
  CLAUDE.md   <- global
CLAUDE.md is the onboarding doc you write once that means every Claude session starts already knowing your project.

PART 4 · CLAUDE.md
LIVE DEMO

Three Levels of Memory — Most Specific Wins

Claude reads all three files and merges them. More specific overrides less specific.
GLOBAL
~/.claude/CLAUDE.md
Every Claude session on your machine
Your personal preferences, writing rules, tools you use across all projects. Private — never committed to git.
PROJECT
./CLAUDE.md
Any session in this project folder
Project context, tech stack, file conventions, hard rules. Committed to git — shared with your team.
SUBFOLDER
./src/CLAUDE.md
Only when working inside that folder
Module-specific overrides. Loads on demand when Claude reads files in that subfolder. Not loaded at launch.
Start with a project CLAUDE.md. Add a global one when you work across multiple projects.

PART 5 · SETTINGS

Two Settings Files — One Shared, One Private

.claude/settings.json
Project settings — commit to git
  • Shared with your whole team
  • Allowed commands everyone can run
  • Denied commands (e.g. never curl *)
  • MCP server config for the project
  • Default permission mode
.claude/settings.local.json
Personal settings — gitignored
  • Only on your machine
  • Your sandbox URLs and test data
  • Personal allow/deny overrides
  • Overrides project settings locally
  • Auto-gitignored — safe to be specific
Use settings.json for team rules, settings.local.json for your personal machine preferences.

PART 5 · SETTINGS

Permissions Let You Pre-Approve What Claude Can Run

  • By default: Claude asks permission before every file edit or command
  • Allow list: pre-approve commands Claude can always run without asking
  • Deny list: hard-block commands Claude can never run in this project
  • Format: Bash(command) or Read(path) or WebFetch(domain:url)
  • First match wins: deny is checked before ask, ask before allow
  • Use /config inside a session to open the settings interface
{
  "permissions": {
    "allow": [
      "Bash(npm run lint)",
      "Bash(npm run test *)",
      "Read(~/.zshrc)"
    ],
    "deny": [
      "Bash(curl *)",
      "Read(./.env)",
      "Read(./secrets/**)"
    ]
  }
}
The deny list is your safety net — block .env files and secret folders before you forget.

PART 6 · MCP

MCP: Claude's Standard Plug for External Tools

MCP stands for Model Context Protocol. It is an open standard that lets Claude Code connect to external tools and data sources.
Jira + Confluence
Create tickets, search issues, publish pages — all from the terminal without copy-pasting
GitHub
Review PRs, create issues, push commits — Claude reads the PR, writes the review
Slack + Notion
Read channel summaries, create Notion pages, post updates — no browser tab switching
Your own data
Connect a database, an internal API, or a custom script. MCP is an open standard — anything goes.
MCP is why Claude Code beats copy-pasting into a chat window — it reads and writes your real tools.

Adding an MCP Server: Four-Step Pattern

Example: adding the Atlassian server (Jira + Confluence)
1
Run the add command in your terminal (outside Claude Code)
claude mcp add --transport http atlassian https://mcp.atlassian.com/v1/mcp
2
Verify it was added
claude mcp list → you should see atlassian listed as 'Needs authentication'
3
Start a Claude session, run /mcp to trigger OAuth login
/mcp → opens your browser, log in with your Atlassian account, authorize
4
Confirm connection
claude mcp list again → atlassian should now show 'Connected'
One command adds a server, /mcp connects it, and then Claude can talk to your tools directly.

Antigravity IDE: Claude Inline in Your Editor

Antigravity IDE supports the extension marketplace, so the Claude Code extension installs without extra configuration.
  • What you get: inline diffs, @-mentions, conversation history in your editor
  • Mac install: Cmd+Shift+X to open Extensions, search Claude Code, click Install
  • Windows install: Ctrl+Shift+X, search Claude Code, click Install
  • After install: Cmd/Ctrl+Shift+P > type Claude Code > Open in New Tab
  • Same CLAUDE.md and settings apply — no duplicate setup needed
  • Prefer the terminal? You do not need the IDE extension at all
IDE vs Terminal
  • IDE: visual diffs, side panel
  • IDE: @-mention files with cursor
  • IDE: plan review as markdown doc
  • Terminal: full command set
  • Terminal: MCP config via CLI
  • Both share history and settings
Start in the IDE if you find terminals intimidating — it gives you visual diffs before Claude edits your files.

PART 4 · CLAUDE.md

CLAUDE.md: What Works vs What Does Not

AVOID
“Format code properly”
Too vague — Claude guesses
DO
“Use 2-space indentation”
Specific — Claude knows exactly
AVOID
“My current priorities are X and Y”
Changes every session — do not put here
DO
“Tech stack: TypeScript, Next.js 14, PostgreSQL”
Stable — belongs in CLAUDE.md
AVOID
“API_KEY=sk-abc123 DATABASE_URL=...”
Secret — never in CLAUDE.md
DO
“Secrets live in .env — never commit to git”
Safe alternative
AVOID
“Here is every edge case and exception...”
Too long — reduces adherence
DO
“Cover the 80%. Trust Claude for edge cases.”
Concise — better followed
Treat CLAUDE.md like a briefing doc for a new hire — facts, rules, context. Not a diary.

GLOSSARY

Terms You'll Hear All Workshop

CLAUDE.md
Persistent project notes Claude reads every session — your briefing doc that never expires.
MCP
Model Context Protocol — the standard that lets Claude talk to external tools like Jira, Figma, and Chrome.
Skill
A markdown file with instructions that loads when you type a trigger phrase — like a saved macro.
Subagent
A separate Claude instance with its own clean context, spun up to handle one focused task.
Plan mode
Read-only mode where Claude shows the plan and waits for your approval before doing anything.
Slash command
A built-in command typed directly inside Claude, starting with /: /help, /clear, /doctor.
Permissions
Pre-approval rules in settings.json that tell Claude what it can run without asking each time.
settings.json vs .local.json
settings.json is shared and committed to git. settings.local.json is personal and stays on your machine only.
Knowing the names makes every other doc readable.

FAQ

Five Questions You'll Be Asked Back at Work

1
Claude Code vs Cursor / Windsurf?
Anthropic frames Claude Code as an agentic coding tool: it reads your codebase, edits files, runs commands. IDE chat tools suggest; Claude Code executes. CLAUDE.md and auto-memory persist across sessions.
2
Is it safe to use Claude with company data and PII?
Anthropic policy: Pro and Max chats and coding sessions are NOT used for training by default. Training requires you to opt in via Privacy Settings or thumbs up / down. Teams and Enterprise add admin controls.
3
Pro vs Max — which subscription?
Pro: $17/mo annual or $20/mo. Max: from $100/mo with 5x or 20x the Pro usage. Both include Claude Code, both run Sonnet and Opus. Start with Pro. Move to Max only if you hit limits.
4
Will my old npm-installed Claude Code conflict with the native install?
Both methods install the same binary. If claude --version points to the old install or you see odd behavior, run: npm uninstall -g @anthropic-ai/claude-code, then reinstall native.
5
Can I use Claude Code without a GitHub account or knowing git?
GitHub is not required. Git for Windows is recommended on native Windows so Claude can use the Bash tool. Without it, Claude Code falls back to PowerShell. Mac, Linux, and WSL do not need it.
